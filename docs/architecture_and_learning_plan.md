# Windows 波段趋势量化提醒系统架构与学习路线

## 1. 系统定位

这是一个本地低频量化提醒系统，不是自动交易机器人。

核心流程：

```text
市场数据 -> 指标/因子 -> 趋势信号 -> 仓位/风控 -> Supervisor审查 -> 网页/邮件 -> 人工决策
```

系统适合日线/周线波段趋势，不适合日内高频。

## 2. 每层做什么

### 数据层

把不同数据源统一成：

```text
date, open, high, low, close, volume
```

短期免费优先：

```text
yfinance -> Stooq -> FMP
```

以后接付费 API 时，只替换数据源，不改策略逻辑。

### 指标/因子层

Python 负责计算所有可重复的指标：

- 均线；
- 动量；
- RSI；
- ATR；
- 相对强度；
- 质量分；
- 未来因子实验收益。

这类事情不要交给 AI，因为 Python 更稳定、可复现、可回测。

### 信号层

信号层把多个指标合成动作：

```text
加仓候选 / 小仓候选 / 观察 / 减仓候选 / 退出候选
```

信号只代表“值得你看”，不是自动买卖指令。

### 仓位与风控层

系统用 LOTS 控制仓位：

```text
可以买股数 = min(目标仓位股数, 风险预算股数)
```

风控规则优先级高于买入信号。

### Supervisor 层

AI 只做最后复核：

- 是否数据质量有问题；
- 是否仓位过大；
- 是否缺止损；
- 是否杠杆 ETF 风险过高；
- 是否需要人工复核。

AI 不负责：

- 每天算指标；
- 回测；
- 止损；
- 自动下单；
- 审查全部股票。

## 3. Windows 正式运行方式

Windows 上建议两部分同时存在：

```text
本地网页服务 + Task Scheduler 定时任务
```

网页服务负责：

- 查看最新报告；
- 手动刷新；
- 录入持仓；
- 记录买入/加仓/减仓/卖出；
- 查看当前持仓和系统信号是否冲突。

计划任务负责：

- 每天美股收盘后运行；
- 生成报告；
- 发 Outlook/SMTP 邮件。

## 4. 因子怎么学

因子就是一个描述股票状态的量化指标。

例子：

- `mom20`: 过去 20 个交易日涨幅；
- `mom60`: 过去 60 个交易日涨幅；
- `rel20`: 20 日相对 QQQ 强弱；
- `rel60`: 60 日相对 QQQ 强弱；
- `rsi14`: 短期涨跌强度。

测因子的目标不是找到“神奇指标”，而是判断：

- 它有没有区分度；
- 它是否降低回撤；
- 它是否只在某几年有效；
- 它是否导致太频繁交易；
- 它是否只对少数股票有效。

## 5. 推荐学习顺序

1. 先跑通每日报告和邮件。
2. 录入真实持仓，观察系统如何提示加仓/减仓。
3. 学会看趋势、动量、相对强度、ATR。
4. 每次只新增或调整一个因子。
5. 用 `factor-test` 看这个因子是否有区分度。
6. 再考虑基本面因子。
7. 最后才考虑 AI 新闻分析或更复杂模型。

## 6. 何时可以接付费 API

付费 API 不是第一步。

只有当你确认以下需求后再接：

- 免费数据源经常限流；
- 回测需要更长历史；
- 你需要更准确的复权价格；
- 你需要稳定 ETF 数据；
- 你要系统每天无人值守稳定发送邮件。

接入后先做数据覆盖测试，不直接改策略。
